「Tomcat 篇」IDEA配置Tomcat

Posted on 2018-10-01 | In Tomcat 篇 |

Tomcat配置

所需条件：
Intellij IDEA 集成开发环境；
JDK；
Tomcat 包（在Tomcat官网上下载对应版本，然后解压即可。如果是Linux操作系统在后续的添加过程中需要对Tomcat包所在的文件夹多执行一步提升文件操作权限的操作。）；

Linux中IDEA中提示Tomcat权限不够的时候在Tomcat文件下的上一层目录下输入以下终端命令即可：sudo chmod -R 777 目录名

步骤：（第一次配置和之后的配置的过程有些不一样，第一次配置的过程稍微多一点，非第一次配置由于以前配置过有些文件已经加载，所以配置步骤更简单）

  1. 创建一个项目
    File -> New Project -> 输入项目名称 -> Fished。

    注：因为不是第一次配置，图中的 Web Application 第一次是没有的

  2. 在创建的项目中创建一个Module
    在上述创建的项目名上单击鼠标右键 -> New -> Module

    输入名称 -> Next -> 勾选 Web Application -> Finish

  3. 创建好Web项目文件，如下：

    以上三步在非第一次配置的时候可以合并成一步：

    在该图中选择JavaEE下的Web Application直接创建Web项目即可，后续的操作都是一样的。

  4. 在创建好的Web项目下的 web 下的 WEB-INF文件夹下创建两个文件夹：clasess 和 lib 。clasess文件用于后续创建的Java文件编译后的输出class文件；lib文件夹用于存放后续操作需要导入的jar包。

    该lib文件夹在后续的操作中，导入需要的jar包之后可能代码中还是会出现没有相应类的错误，此时可以手动操作，在lib文件夹上鼠标单击右键，在弹出的菜单中选择：Add as Library… 即可。也可以在项目结构中添加：File -> Project Structure(快捷键 Ctrl + Shift + Alt + S) -> Libraries -> + 号 -> Java，然后在弹出的菜单中通过路径选择lib包即可。

  5. 配置文件夹路径
    File -> Project Structure (快捷键：Ctrl + Shift + Alt + S) -> 选择Module ：
    选择 Paths -> 选择”Use module compile output path” -> 将Output path和Test output path都选择为上述创建的classes文件夹

    然后选择Dependencies -> 点击右边的“+”号 -> 选择1 “Jars or Directories”

    选择刚刚创建的lib文件夹 -> 选择“jar directory” -> OK

  6. 配置Tomcat
    菜单 Run -> Edit Configuration (或者点击下图中的下拉框中的 Edit Configuration)：

    点击“+”号 -> 选择“Tomcat Server” -> 选择“Local”

    在”Name”处输入新的服务名，点击”Application server”后面的”Configure…”，弹出Tomcat Server窗口，选择本地安装的Tomcat目录 -> OK
    在”Run/Debug Configurations”窗口的”Server”选项板中，取消勾选”After launch”（也可不取消，表示是否要在启动Tomcat后自动打开浏览器），设置”HTTP port”和”JMX port”（默认值即可），点击 Apply -> OK， 至此Tomcat配置完成。

  7. 在Tomcat中部署项目
    Run -> Edit Configurations，进入”Run/Debug Configurations”窗口 -> 选择上述创建的Tomcat -> 选择Deployment -> 点击右边的“+”号 -> 选择Artifact ->选择web项目 -> Application context（可填可不填） -> Apply

  8. 编辑index.jsp文件，并运行

Tomcat 目录

bin：脚本目录
    启动脚本：startup.bat
    停止脚本：shutdown.bat
conf：配置文件目录 (config /configuration)
    核心配置文件：server.xml
    用户权限配置文件：tomcat-users.xml
    所有web项目默认配置文件：web.xml
lib：依赖库，tomcat和web项目中需要使用的jar包
logs：日志文件.
    localhost_access_log..txt tomcat记录用户访问信息，星表示时间。
    例如：localhost_access_log.2016-02-28.txt
temp：临时文件目录，文件夹内内容可以任意删除。
webapps：默认情况下发布WEB项目所存放的目录。
work：tomcat处理JSP的工作目录。
