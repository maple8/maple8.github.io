HashMap

HashMap集合和HashSet集合类似，都是基于哈希表的集合,其实HashSet集合的底层是HashMap集合。集合通过哈希码可以对其内部的映射关系进行快速查找。HashMap集合是针对键的，也就是侧重点是键，下面的特点也都是针对键而言的。
HashMap集合的特点：

  • 存取无序
  • 无索引
  • 键唯一，键值对的映射关系也唯一：不能出现相同的键，如果想集合中添加相同的键但是对应不同的值，那么会将该键的原始值弹出。

HashMap集合的遍历

  1. HashMap的键遍历keySet()：
    通过keySet()方法将HashMap集合中的所有的键组成Set集合返回，然后利用Set集合的方法生成迭代器，通过迭代器来实现遍历。

代码冗余优化版：

  1. HashMap的值遍历values()：
    通过values()方法将HashMap集合中的所有的键组成Collection集合返回，然后利用Collection集合的方法生成迭代器，通过迭代器来实现遍历。

代码冗余优化版：

  1. HashMap键值对遍历entrySet()：
    通过entrySet()方法将HashMap集合中的所有键值对都组合成一个元素放入Set集合中，该集合的每一个元素都是一个键值对，然后通过Set集合的方法获取迭代器，遍历该集合便可得到键值对元素，使用entry.getKey()方法和entry.getValue()方法可以获得键值对中的键和值。

HashMap集合的嵌套

HashMap集合的嵌套就是HashMap集合中的键还是HashMap集合，也就是说主要是针对键，这也是HashMap集合的侧重点，该集合本身的特性就是针对键的。

HashMap集合实例

充分利用了HashMap集合的键唯一性，同时又需要记录次数，从算法的时间复杂度和空间复杂度来说，Map集合是最好的选择，然后又要尽量的考虑代码的效率，HashMap是最好的。

需求：统计字符串中每个字符出现的次数，如：aaaaaabbbbbbbccccccccdddddddd
a=6
b=7
c=8
d=8

三种Map集合: HashMap,LinkedHashMap,TreeMap
效率最高的是 HashMap

HashMap和Hashtable的区别
共同点: 底层都是哈希算法,都是双列集合
不同点一:

1. HashMap是线程不安全的,效率高,JDK1.2版本
2. Hashtable是线程安全的,效率相对低,JDK1.0版本
不同点二:
1. HashMap可以存储null键和null值

2. Hashtable不可以存储null键和null值,会出现空指针异常
