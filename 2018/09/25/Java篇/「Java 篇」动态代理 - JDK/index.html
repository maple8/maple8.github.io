「Java 篇」动态代理 - JDK

Posted on 2018-09-25 | In Java 篇 |

使用动态代理的目的：对目标对象（被代理对象）中的方法进行增强，功能类似于Python中的装饰器。

动态代理

动态代理的意义在于生成一个占位（又称代理对象），来代理目标对象，从而控制对目标对象的访问。动态代理不用手动编写代理对象，不需要一一编写与目标对象相同的方法，这个过程，在运行时的内存中动态生成代理对象（字节码对象级别的代理对象）。
可以用一张图来表示代理模式的含义：

在调用者调用目标对象之前产生一个代理对象，而这个代理对象需要和目标对象建立代理关系，所以代理必须分成两个步骤：

  1. 代理对象和目标对象建立代理关系；
  2. 实现代理对象的代理逻辑方法；

Java中有多种动态代理技术，常用的有两种：JDK动态代理和CGLIB动态代理，此处介绍的是利用JDK动态代理。
在Spring中会混合使用这两种动态代理技术，优先使用JDK动态代理，如果没有接口的时候就使用CGLIB动态代理。
特点：

  1. JDK动态代理技术是java.lang.reflect.*包提供的方式，被代理对象必须要实现一个接口，才能产生一个代理对象，如果没有接口将不能使用JDK动态代理。
  2. JDK动态代理中的代理对象和被代理对象两者实现了同一个接口。
