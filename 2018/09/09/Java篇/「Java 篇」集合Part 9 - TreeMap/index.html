TreeMap

TreeMap集合底层是二叉树实现的，可以对键进行排序，所以不允许键对象是null，否则就会抛出空指针异常。TreeMap集合也是针对键的。
TreeMap集合的特点：

  • 键唯一性
  • 键排序
  • 无索引

TreeMap集合的使用

TreeMap集合默认会对键进行排序，排序的底层是通过compareTo()方法，String类型默认已经重写了compareTo()方法，按照字典的顺序进行排序。

其他的常用方法和HashMap集合一样

TreeMap集合存储自定义对象

和TreeSet集合一样，自定义类对象如果想要实现该集合的特性就要实现Comparable接口，并重写compareTo()方法。

自定义类：

通过匿名内部类创建比较器Comparator的子类对象，然后进行compareTo()方法的重写实现指定的顺序排序键。该方法适合所有类型的对象，自定义类对象，基本类型对象的封装形式等等。
