HashSet

HashSet集合是哈希表支持的，底层是HashMap的操作。
HashSet的特点：

  • 存取无序：不保证存储和获取元素的顺序是一致的
  • 无索引：无法通过索引获取元素，获取元素只能通过迭代的方法
  • 不能存储重复元素：底层是通过hashcode方法和equals方法对各个对象进行比较，然后决定是否存取，所有基本数据类型的引用数据类型都重写了这两个方法，所以在作为元素进行添加的时候能够自动进行比较，如果是自定义类的实例，必须要开发者在定义类的时候重写这两个方法，否则就不能实现不重复存储的特性。

HashSet存储自定义数据类型
自定义数据类型对象作为元素添加到HashSet集合当中，默认是不会满足HashSet的特点，主要就是不能实现不重复存储的特点。如果要实现该特点，必须在定义自定义类的时候，重写hashcode方法和equals方法。

hashcode()方法和equals()方法如何实现不重复存储呢？
hashCode()方法在创建对象的时候就会自动调用，开发者创建的对象要保存在堆内存中,既然要保存在内存中,那么地址就是必不可免的，而这个地址值就是通hashCode()方法得来的哈希值。而equals()方法是在hashCode()执行之后，如果多个对象的hashCode值相等就自动调用equals()方法进行判断，判断这两个hashCode值相同的对象是不是相同的对象，返回值为false表示不是相同的对象就存入，返回为true就表示是相同的对象就不存。所以，开发者在创建自定义的HashSet的类的时候就必须重写。
