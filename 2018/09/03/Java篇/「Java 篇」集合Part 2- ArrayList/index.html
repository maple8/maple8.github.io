ArrayList

ArrayList集合底层是数组实现，所以查找和修改操作速度快，删除和增加操作速度慢，因为数组在内存中是连续连续存储的，查找和修改操作可以直接通过索引快速实现，时间复杂度为 O(1)，但是对于删除和增加操作，每次执行这样的操作，数组的得到长度都会发生变化，但是又要保证内存的连续存储，如果在数组中间删除或是增加了一个元素，那么其后面的所有元素都需要执行一次移位操作来保证内存的连续性，最好的情况是在数组的末尾执行删除或是增加操作，时间复杂度是 O(1)，最坏的情况是在数组的开头执行删除或是增加操作，那么所有的数组元素都要执行依次一位操作，所以这样的一次删除或是增加操作的时间复杂度就是 O(n).
ArrayList底层在内存中的存储方式决定了使用场景：

  • 查找或修改操作频繁的任务
  • 增、删、改、查 操作都频繁的任务

ArrayList特点：

  • 顺序存储：能按照元素的存入顺序进行保存和读取
  • 有索引：可以通过索引获取元素
  • 可以存储重复元素

List的遍历

List集合有两种迭代器，一种是 Iterator，一种是 ListIterator 。这两种迭代器的工作方式都是一样的，但是功能上有点不同。Iterator迭代器只能是单纯的迭代，加上删除操作，否则会抛出异常。ListIterator迭代器则可以在遍历集合的过程中对集合进行增、删、改等操作。
