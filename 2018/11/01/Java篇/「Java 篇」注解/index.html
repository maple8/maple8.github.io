注解作用：

  1. 注释：在阅读程序时给程序员看的，有助于提高程序的可读性。
  2. 注解：给jvm看的，能够准确地执行底层的注解解析代码，实现注解的功能。

注解的应用：
注解在目前最主流的应用：
代替配置文件，尤其是xml文件。

注解相对于配置文件在开发中的优缺点：

  • 注解优点：开发效率高，成本低
  • 注解缺点：耦合性大，并且不利于后期维护

jdk5提供的注解

  1. 方法重写：常用于对接口中的方法进行重写标志

  2. 过时：表示调用注解的对象已经过时了

  3. 压制警告：表示在调用注解对象内部消除警告提示

jdk8提供的注解

  1. 重复注解：可以在同一个对象上多次调用用一个注解

这个就是元注解，它代表修饰注解的注解，其作用就是限制定义的注解的特性。
@Target
代表注解修饰的范围：类上使用，方法上使用，字段上使用

  • FIELD：字段上可用此注解
  • METHOD：方法上可以用此注解
  • TYPE：类/接口上可以使用此注解

@Retention
代表注解的可见级别，有以下三种：

  • SOURCE: 注解在源码级别可见
  • CLASS：注解在字节码文件级别可见
  • RUNTIME：注解在整个运行阶段都可见

    要想解析使用了注解的类 ， 那么该注解的Retention必须设置成Runtime，因为注解的解析是基于反射技术的，而反射的作用区域就是运行时环境。
