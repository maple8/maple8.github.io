JDBC

JDBC（Java DataBase Connectivity），Java数据库连接，由SUN公司推出的Java访问数据库的标准规范（接口），是连接数据库和Java应用程序的纽带。其有如下特点：

  1. JDBC 是一种用于执行SQL语句的Java API；
  2. JDBC 可以为多种关系数据库提供统一的访问入口；
  3. JDBC 由一组Java工具类和接口组成；

导入驱动包

  1. 下载JDBC的驱动jar包；
    以下是 mysql-connector-java-5.1.39-bin.jar 的百度网盘下载链接：
    https://pan.baidu.com/s/1j7Wr4Vz9d0VZ_WFBmIP7GQ 提取码: u9jr
  2. 在IDEA中需要使用MySQL的项目中创建一个文件夹，一般命名为lib/libs(eclipse中是这样命名的)，然后将下载的驱动jar包拷贝到文件夹中；
  3. 将lib文件夹中的jar包添加到项目里，类似eclipse的add to build path功能（添加成功之后会出现一个牛奶瓶的logo）。
    Fille->Project Structure (或快捷键 Ctrl+Alt+Shift+S),弹出图示菜单：

    点击最右边的+号，点击 1.JARs or directories，在弹出的菜单中选择需要导入的jar包：
  4. OK -> Apply -> OK，JDBC的驱动jar包就导入成功了。

注册驱动

注册驱动代码：

  1. JDBC规范定义驱动接口：java.sql.Driver，MySQL驱动包提供了实现类：com.mysql.jdbc.Driver；
  2. 通常开发的时候使用Class.forName()将一个使用字符串描述的驱动类。 使用Class.forName()将类加载到内存，该类的静态代码将自动执行。
  3. DriverManager工具类。提供注册驱动的方法 registerDriver()，方法的参数是java.sql.Driver，所以我们可以通过如下语句进行注册DriverManager.registerDriver(new com.mysql.jdbc.Driver());

以上方法不推荐使用，因为有两方面的不足：

  1. 硬编码，后期不易于程序拓展和维护，如果更换数据库的类型，则源码都要改变；
  2. 驱动被两次注册；

com.mysql.jdbc.Driver 源码中Driver类会“主动”将自己进行注册，因为注册代码是在静态代码块中。

建立连接

建立连接代码：

建立连接需要方法 DriverManager.getConnection(url, username, password) ,三个参数分别表示：
  url：需要连接数据库的位置（网址）；
  username：数据库用户名（一般都是root）；
  password：数据库的登录密码；

url 比较复杂，下面是MySQL的url：

JDBC规定url的格式由三部分组成，每个部分中间使用逗号分割。
  第一部分：jdbc，固定的；
  第二部分：数据库类型名称；
  第三部分：由数据库厂商规定的，MySQL的第三部分分别是 数据库服务器的IP地址(localhost)、端口号(3306)、以及 数据库名称(chapter4) 组成；

url参数

useUnicode 参数： 指定这个连接数据库的过程中，使用的字节集是Unicode字节集；
useSSL 参数版： 指明是否进行SSL连接；

  如果没有这个参数，Java使用mysql-jdbc连接MySQL出现如下警告：  Establishing SSL connection without server’s identity verification is not recommended. According to MySQL 5.5.45+, 5.6.26+ and 5.7.6+ requirements SSL connection must be established by default if explicit option isn’t set. For compliance with existing applications not using SSL the verifyServerCertificate property is set to ‘false’. You need either to explicitly disable SSL by setting useSSL=false, or set useSSL=true and provide truststore for server certificate verification.

characterEncoding 参数： 指定Java程序连接数据库的过程中，使用的字节集编码为UTF-8编码；

注意：MySQL中的语法是utf8，而不是utf-8.
