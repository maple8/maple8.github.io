「Java 篇」System.gc( )

Posted on 2019-04-10 | In Java 篇 |

        Java具有内存管理机制，让开发者在开发过程中不用考虑对象的内存分配问题，专注于项目的逻辑功能开发。但是，这一功能也是Java效率低下的一个重要原因。Java中的内存分配是随着new一个新的对象来实现的，这一过程底层都是JVM自动实现的。为了避免JVM自动管理内存导致的效率低问题，也还是有一些“改进”内存回收的机制的方法，其中最显眼的就是这个System.gc()函数。

System.gc()函数的作用

有些人认为这个方法是让虚拟机回收不用的对象，可事实并非如此。
System.gc()函数的作用：提醒虚拟机开发者希望进行一次垃圾回收。但是它不能保证垃圾回收操作是否一定会执行，在什么时候执行。
这些都是取决于具体的虚拟机的，而且不同的虚拟机有不同的对策。也就是说开发者对于对象的回收操作没有绝对的控制权。

JVM回收对象的准则

在Java中是通过可达性分析算法来判定一个对象是否需要被回收。该算法的基本思路就是：通过一系列称为“GC roots”的对象作为起点，从这些节点开始向下搜索，搜索所走过的路径被称为引用链，当一个对象到GC roots没有任何引用链相连时，该对象就是需要回收的。
Java中可作为GC roots的对象：

  • 虚拟机栈（栈帧中的本地变量表）中引用的对象
  • 方法去中的类静态属性引用的对象
  • 方法去中常量引用的对象
  • 本地方法栈中JNI（一般说的Native方法）引用的对象
