硬件平台:
Ubuntu 16.04
Git 2.7.4
Intellij IDEA Ultimate

Git 分支的使用

分支主要是用于阶段性的发布开发项目的大版本，比如：完成第一阶段的1.0版本开发之后，需要将该版本推送到远程仓库，然后还需要再继续开发版本2.0，增加某些功能，为了保证这些大版本之间不会互相影响，采用分支，其实我们默认的就是主分支master。

  1. 1.0 版本开发完成之后，将其发布到远程仓库
    在项目名称上右键 –> Git –> Repository –> Branches


    使用快捷键：Ctrl + K 进行commit；Ctrl + Shift + K 进行push，在远程仓库中显示如下：

    这样就将一个大版本1.0发布到码云上了。
  2. 切换到主分支继续进行2.0版本的开发

    添加了一个新的类
    此时如果发现版本1.0有出现了bug，那就要切换到10分支进行修复，修复bug之后再将修复后的1.0版本发布

    然后切换会主分支继续开发，此时由于版本1.0已经做了修复bug的操作，而现在是在原始有bug的1.0版本上继续开发，需要将当前开发的分支和远程仓库的1.0版本进行合并以更新当前分支中的1.0版本存在的bug:

    此时就可以看见1.0修复的代码已经更新到当前分支中了

    将合并之后的代码推送到远程仓库中，会出现一行(Merge remote-tracking branch ‘remotes/origin/1.0’)


    这样就实现了分支的作用。

    查看历史提交记录

    Git 的所有提交记录都是可以查看的：
    在项目名称上右键 –> Git –> Show History或者点击右上角的小时钟logo（需要先选中项目或文件）

对比文件的不同版本

Git 会记录一个文件的所有提交版本，可供后续查看，以对比显示做出了那些修改
先选中文件 –> 鼠标右键 –> Git –> Compare With –> 选择需要对比的版本

版本回退

如果出现了误操作，并且已经提交到了远程仓库中了，可以使用版本回退来获取误操作之前的代码：
先选中出现误操作的文件 –> 点击右上角的小时钟logo查看所有历史版本 –> 选中前一个版本右键 –> Get
